# 08. 未決事項・リスク管理

## 未決事項

| ID | 論点 | 初期案 | 判断が必要な理由 |
|---|---|---|---|
| D-001 | 認証方式 | Cookie認証 + Antiforgery | 現行Form Login + CSRFに近いが、JWT案との比較が必要 |
| D-002 | DBアクセス方式 | DapperまたはEF Core + 明示SQL | 現行はSQL明示。Migration管理とのバランスが必要 |
| D-003 | 帳票ライブラリ | ClosedXMLまたはNPOI | テンプレート互換、ライセンス、書式再現性を確認 |
| D-004 | 月次帳票出力方式 | 初期は同期、将来非同期 | `PROCESSING` ステータスをどう活かすか決める |
| D-005 | ファイル保存先 | 設定可能な共有保存領域 | 複数台構成、バックアップ、権限制御に影響 |
| D-006 | React状態管理 | React Context + TanStack Query | 画面横断状態が増える場合は追加ライブラリ検討 |
| D-007 | 現行未実装機能 | 初期スコープ外 | 編集、削除、承認、CSV、ページング等を含めるか |
| D-008 | 画面デザイン準拠範囲 | 現行機能再現を優先 | `docs/designs/` には現行より高機能な要素がある |
| D-009 | データ移行方式 | 停止時間中の一括移行 | 並行稼働や差分移行が必要か判断 |
| D-010 | 帳票履歴のパス移行 | 新保存先に合わせて再計算 | 現行 `file_path` をそのまま使えるとは限らない |

## リスク一覧

| ID | リスク | 影響 | 対策 |
|---|---|---|---|
| R-001 | Oracle固有SQLの変換漏れ | 検索、集計、帳票結果が不一致 | SQL棚卸し、API/DB責務分離、比較テスト |
| R-002 | 採番シード調整漏れ | 登録時にID衝突 | 移行後最大ID確認、IDENTITY seed調整 |
| R-003 | 日本語文字化け | 画面、帳票、履歴の表示不正 | `nvarchar` 使用、移行後サンプル確認 |
| R-004 | 認可漏れ | 他人の実績や帳票を参照可能 | API側Policy、ユーザー別テスト |
| R-005 | CSRF/CORS設定不備 | 不正リクエストまたは画面利用不可 | 認証方式別のセキュリティテスト |
| R-006 | Excel書式差分 | 業務帳票として受け入れ不可 | 現行Excelとのセル値/書式比較 |
| R-007 | ファイルパストラバーサル | 保存領域外ファイル漏えい | ファイル名サニタイズ、real path検証 |
| R-008 | 長時間帳票出力 | タイムアウト、二重実行 | 非同期化検討、二重送信防止 |
| R-009 | ロールバック不能 | 障害時に現行へ戻せない | 切替直前バックアップ、戻し手順リハーサル |
| R-010 | 仕様追加の混入 | 移行スケジュール遅延 | 現行再現と追加改善を分けて管理 |

## 重点チェックリスト

- 現行の全画面、全API候補、全帳票、全権限を棚卸しした。
- Oracle固有の型、関数、採番、制約をSQL Server向けに変換した。
- SQL Server移行後の件数、FK、制約、主要集計が一致した。
- 一般ユーザーは自分の作業実績、帳票履歴、帳票ファイルだけ参照できる。
- 管理者の検索範囲、帳票出力範囲が現行仕様と一致している。
- React側の表示制御だけでなく、API側で認可している。
- CSRF、CORS、Cookie、SameSite、HTTPS設定を検証した。
- 現行Excelと新Excelのセル値、書式、ファイル名、履歴が一致した。
- 帳票再ダウンロードで保存先外ファイルを読めない。
- 接続文字列、パスワード、秘密情報をリポジトリに置いていない。
- リリース前バックアップとロールバック手順を検証環境で確認した。
- 未決事項に担当者、期限、判断結果を記録している。

## 初期実装に入る前の確定事項

2. DBアクセス方式。
3. 帳票ライブラリ。
4. SQL Server版DDLと採番方式。
5. 帳票保存先。
6. 移行対象機能の凍結範囲。
7. リリース方式と停止時間。
