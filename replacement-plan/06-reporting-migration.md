# 06. 帳票移行計画

## 現行仕様

| 項目 | 内容 |
|---|---|
| 帳票ID | `MONTHLY_WORK_REPORT` |
| 帳票名 | 月次作業報告書 |
| 形式 | Excel `.xlsx` |
| テンプレート | `src/main/resources/templates/monthly-report-template.xlsx` |
| 生成方式 | Apache POI `XSSFWorkbook` でテンプレートを読み込み、固定セルと明細行へ値を設定 |
| 保存先 | `generated-reports/{YYYYMM}/` |
| 履歴 | `report_output_histories` に `PROCESSING`, `SUCCESS`, `ERROR` を保存 |
| 再ダウンロード | 成功済み履歴の保存ファイルを返却 |

現行テンプレートは1シート構成で、固定セル、分類別明細テンプレート行、日別明細テンプレート行を持ちます。C#移行時は、セル位置、行追加、スタイルコピー、印刷設定をテンプレート互換の検証対象にします。

## 出力データ

| 区分 | 内容 |
|---|---|
| 基本情報 | 対象年月、社員名、部署名、作成日 |
| サマリー | 総作業時間、稼働日数、平均作業時間 |
| 分類別集計 | 作業分類名、合計時間、比率 |
| 日別明細 | 作業日、曜日、プロジェクト名、作業分類名、作業時間、作業内容 |

## C#側ライブラリ候補

| 候補 | 特徴 | 注意 |
|---|---|---|
| ClosedXML | Excel操作が比較的簡潔。初期PoCの第一候補 | テンプレート行コピーや書式互換を検証 |
| EPPlus | 機能が多い | ライセンス条件を確認 |
| NPOI | Apache POIに近い思想 | APIの使い勝手、保守性を確認 |
| Open XML SDK | Microsoft公式、低レベル制御 | 実装量が増えやすい |

初期方針は、まずClosedXMLでテンプレート差し込みPoCを作り、現行テンプレートと代表データで値、書式、ダウンロード、履歴更新まで検証します。採用前に各ライブラリの最新ライセンスと商用利用条件を確認します。

## 移行方針

1. 現行テンプレートをそのまま読み込めるか検証する。
2. 固定セルの書き込み位置を現行 `ExcelReportService` から対応表化する。
3. 分類別集計、日別明細の行追加と書式コピーをC#で再現する。
4. 現行と新出力のセル値、書式、印刷範囲、幅、高さを比較する。
5. 履歴ステータス更新、ファイル保存、再ダウンロードをAPIへ移行する。

## 履歴保存

| タイミング | ステータス | 処理 |
|---|---|---|
| 出力開始 | `PROCESSING` | 履歴IDを確保し、仮ファイル名を保存 |
| 出力成功 | `SUCCESS` | 正式ファイル名、保存先、エラーなしへ更新 |
| 出力失敗 | `ERROR` | エラーメッセージを保存し、作成途中ファイルを削除 |

## ファイル保存と安全対策

- 保存ファイル名は社員名などの入力由来文字列を必ずサニタイズします。
- 保存先は設定値化し、APIサーバーの作業ディレクトリに暗黙依存しないようにします。
- 再ダウンロードでは履歴IDからDBを引き、保存領域配下の実ファイルだけを返します。
- `..`、絶対パス、制御文字、パス区切り文字を含むファイル名を拒否します。
- `SUCCESS` 以外の履歴はダウンロード不可にします。
- HTTPレスポンスは `.xlsx` のMIME typeを設定し、`Content-Disposition` で日本語ファイル名を安全に返します。

## 同期/非同期の判断

| 案 | 内容 | 向いている状況 |
|---|---|---|
| 同期ダウンロード | `POST /api/monthly-reports/export` がExcelを直接返す | データ量が少なく、処理が短い |
| 非同期出力 | 出力要求で履歴を作り、画面で状態更新後にダウンロード | データ量増加、タイムアウト懸念、監査重視 |

現行はダウンロードと履歴保存を同時に行っています。リプレイス初期は現行互換の同期方式を基本にしつつ、`PROCESSING` ステータスを活かせる非同期化を拡張候補にします。

## 検証観点

| 観点 | 確認内容 |
|---|---|
| セル値 | 対象年月、社員名、部署名、合計、平均、分類別、明細が一致 |
| 書式 | フォント、罫線、表示形式、日付、小数、幅、高さ |
| 明細行 | 0件、1件、多件、分類数増加時の行位置 |
| ファイル | ファイル名、拡張子、MIME type、ダウンロード名 |
| 履歴 | `PROCESSING` から `SUCCESS` / `ERROR` への更新 |
| 権限 | 一般ユーザーが他人の帳票を出力・取得できない |
| 異常系 | テンプレートなし、保存先なし、ファイル削除済み、出力中例外 |
| 環境差 | Windows/Linuxのパス区切り、禁止文字、日本語ファイル名 |

## リスク

- Excelライブラリ変更により、見た目の微差が出る可能性があります。
- 既存テンプレートのセル位置に依存しているため、テンプレート改修時に壊れやすいです。
- ファイル保存先がローカルディスクの場合、複数台構成やコンテナ構成で共有設計が必要です。
- 長時間出力時にHTTPタイムアウト、二重クリック、重複履歴が発生する可能性があります。
